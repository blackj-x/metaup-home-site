# MetaUp 母站 SEO 优化执行计划（每周小步版）

**目标**：通过持续小步优化，提升母站内容厚度、E-E-A-T 和搜索可见度，帮助根域名通过 AdSense 审核，同时为子站引流。重点放在母站原创内容 + 站群动态 + 双语支持。

**原则**：
- 每周只做 1-3 个小任务，避免大批量发布。
- 优先高影响、低工作量的事（标题、动态、链接、内链）。
- 站群动态每周至少更新 1 条（真实 + 趋势驱动）。
- 内容生产均匀分散：每 2 周 1 篇新文章（中英错开）。
- 每 2 周用 Google Trends + GSC 复盘一次。
- 中文优先（主力流量），英文同步但可慢半拍。
- 所有改动以数据文件为主（`src/data/`），保持架构干净。
- 每周改完后 `npm run build` 检查 + 本地预览。

**工具准备**（Week 1 完成）：
- Google Search Console（GSC）：验证 metaup.pro，提交 sitemap。
- Google Trends：每周监控 “鹅鸭杀”、“致命公司” + 新角色/怪物。
- 浏览器书签：Trends、GSC、Search Console 性能报告。

## Phase 1: 基础打底（Week 1-3）

### Week 1: 审计 + 快速赢 + 动态启动
**任务**：
1. 跑 GSC：
   - 验证站点。
   - 提交最新 sitemap。
   - 记录当前收录数和查询词。
2. 首页 + /en/ 描述和 hero 文案微调（加入 1-2 个核心意图词，如 “原创攻略”、“可信数据”）。
3. 刷新站群动态（`src/pages/index.astro` 和 `src/pages/en/index.astro`）：
   - 扩展到至少 8-10 条。
   - 日期覆盖 2026-05 ~ 现在。
   - 加 2-3 条“真实感”更新（例如“根据玩家反馈修正 XX 数值”）。
4. 用 Trends 跑一次：
   - “鹅鸭杀” vs “致命公司”。
   - 记录 3-5 个上升相关词（角色、地图、补丁等）。
5. 列出当前所有页面标题（用 `npm run build` 后检查 dist/）。

**产出**：GSC 截图 + Trends 笔记 + 更新后的动态列表。

### Week 2: 标题/元数据优化 + 技术小修
**任务**：
1. 优化现有 4 篇中文 + 6 篇英文编辑文标题 + meta description：
   - 示例：`如何真正读懂游戏更新说明` → `游戏补丁说明怎么看？新手避坑实用方法`
   - 每篇加 1 个长尾（“可信攻略”、“E-E-A-T” 不硬塞，自然放）。
2. 在 `src/layouts/Base.astro` 补全双语 hreflang（中英互为 alternate）。
3. 每篇文章底部自然加 2-3 个内部链接（指向子站具体页 + 其他编辑文 + FAQ）。
4. 给 FAQ 页加 1 个 schema（FAQPage）简单示例（可在 Article 布局附近加）。
5. 站群动态再加 1 条（可以是本周优化的记录）。

**产出**：优化后的标题列表 + 内部链接示例。

### Week 3: 母站核心页结构 + E-E-A-T 强化
**任务**：
1. 优化首页、/articles、/en/ 三个列表页的 H1/H2 和描述。
2. 在 About 和 FAQ 页加强真实描述：
   - 增加“版本跟踪”“玩家反馈闭环”“每周更新”等。
3. 检查双语 canonical 是否正确。
4. 用 Trends 复盘一次（对比 Week 1）。
5. 站群动态保持每周 1 条。

## Phase 2: 内容增强 + 集群（Week 4-8）

**内容生产节奏**：每 2 周产出 1 篇新文章（先中文，后英文，或交替）。主题来自 Trends + 现有系列延伸。

### Week 4: 现有内容集群 + 1 篇新文启动
**任务**：
1. 把“补丁说明”“靠谱判断”“维护”三篇文章互相内链 + 都链到 /resources。
2. 规划/开始写 1 篇新文章（推荐主题：Trends 里上升的新角色或“XX 机制自查清单”）。
   - 只做中文版。
   - 用 `src/data/articles.ts` 添加元数据 + 新建 `src/pages/articles/xxx.astro`。
3. 站群动态 + 1 条（新文章相关）。

### Week 5: 完成新文 + 英文同步
**任务**：
1. 完成 Week 4 新文章中文版，部署测试。
2. 补英文版（可稍短，侧重不同角度）。
3. 在新文章里自然回答 1-2 个“人们也搜索”问题。
4. 更新首页编辑精选区块文案。
5. 站群动态 + 1 条。

### Week 6: 内链大扫荡 + 趋势内容
**任务**：
1. 全局内链检查（母站 → 子站具体高意图页；子站 about 页反链母站 FAQ + 编辑精选）。
2. 根据 Trends 写/规划第 2 篇新文章（例如新地图或“团队沟通原则”延伸）。
3. 优化 Resources 页，加入 1 个实用工具清单。
4. 站群动态 + 1 条。

### Week 7: 第 2 篇新文 + 母站发现强化
**任务**：
1. 发布第 2 篇新文（中英）。
2. 首页 hero 和子站卡片文案微调（突出“可信 + 原创”）。
3. 在 1-2 篇编辑文里加 FAQ 跳转。
4. 站群动态 + 1 条（可以是子站小更新）。

### Week 8: 中期复盘
**任务**：
1. GSC 复盘：收录、点击、查询词变化。
2. Trends 复盘 + 决定下一批主题。
3. 补充 1 篇小内容（可以是 FAQ 扩展或 Resources 新条目）。
4. 站群动态保持节奏。
5. 列出 Week 9-12 优先级调整。

## Phase 3: 深化 + 可持续循环（Week 9-12+）

### Week 9: 技术收尾 + schema
**任务**：
1. 给重要文章页加 Article + HowTo schema（在 layouts 里统一处理）。
2. 检查移动端体验和 Core Web Vitals（GSC 里看）。
3. 规划第 3 篇新文章。
4. 站群动态 + 1 条。

### Week 10: 第 3 篇新文 + 链接强化
**任务**：
1. 发布第 3 篇。
2. 母站所有编辑文至少链 3 个子站具体内容。
3. 更新 About 页，增加“站群动态历史”小节。
4. 站群动态 + 1 条。

### Week 11: 趋势驱动 + 小批量优化
**任务**：
1. 根据最新 Trends 出 1 篇小内容（或现有页更新）。
2. 优化 2-3 个老页面标题/描述（基于 GSC 新查询）。
3. 站群动态 + 1 条。

### Week 12: 完整复盘 + 下周期计划
**任务**：
1. 全面复盘 12 周：
   - 内容产出量
   - GSC 数据
   - Trends 匹配度
   - 站群动态更新频率
2. 制定下一个 12 周计划（循环）。
3. 站群动态 + 1 条（总结性）。
4. （可选）准备 1 篇“AdSense 内容厚度”自查笔记。

## 长期机制（12 周后进入循环）

- **每周固定**：
  - 站群动态至少 1 条真实更新。
  - 10 分钟 Trends 扫描。
  - `npm run build` + 预览检查。
- **每 2 周**：
  - GSC + Trends 复盘。
  - 决定是否加新文章或优化现有。
- **每月**：
  - 检查子站反链情况。
  - 评估 AdSense 准备度（内容页数、更新历史、E-E-A-T 信号）。
- **内容主题来源**：
  - 游戏大更新 → 立即跟进 1 篇。
  - Trends 上升词 → 转化为长尾文章。
  - 玩家反馈（邮箱）→ 直接转化成内容。
  - 现有系列延伸（“沟通原则” → “新手 vs 老手团队差异”）。
